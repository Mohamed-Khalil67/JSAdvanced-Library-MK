                            EX 01
                            Load products from localStorage
                        

                        
Reading persisted data on page load

                    

                

                
▾

            

            

                
What it does

                
When the page first loads, reads any previously saved products from the browser's
                    localStorage. If nothing is stored yet (first visit), it defaults to an empty array so
                    the app doesn't crash.


                
The Code

                
function loadProducts() {
  products = Utils.loadFromStorage(STORAGE_KEY, []);
}


                
Utils Functions Used

                

                    

                        Utils.loadFromStorage(key, fallback)
                        Reads & JSON.parses a value from localStorage. Returns fallback if key
                            doesn't exist.
                    

                


                
💡 How loadFromStorage works internally: It calls
                    localStorage.getItem(key). If the result is null (key doesn't exist), it
                    returns the fallback. Otherwise it calls JSON.parse() on the raw string — because
                    localStorage only stores strings, not real arrays/objects.


                
Key Concepts

                

                    Default parameter (fallback = [])
                    localStorage
                    JSON.parse
                    Utility reuse
                


                
Likely Instructor Questions

                

                    

                        
What happens on a brand new first
                                visit?▾

                        
localStorage has nothing stored under STORAGE_KEY, so
                            getItem() returns null. The function detects this and returns the
                            fallback value [], so products starts as an empty array.

                    

                    

                        
Why do we need JSON.parse
                                here?▾

                        
localStorage can only store strings. When we saved the array earlier, it
                            was converted to a JSON string via JSON.stringify. So when loading it back, we
                            must parse it back into a real JavaScript array with JSON.parse.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 02
                            Persist products to localStorage
                        

                        
Saving data so page refresh doesn't lose it

                    

                

                
▾

            

            

                
What it does

                
Every time the products array changes (add, edit, or delete), this function saves
                    the entire array to localStorage. Without it, all data disappears on page refresh.


                
The Code

                
function persistProducts() {
  Utils.saveToStorage(STORAGE_KEY, products);
}


                
Utils Functions Used

                

                    

                        Utils.saveToStorage(key, value)
                        JSON.stringifies a value and stores it in localStorage under the given
                            key.
                    

                


                
💡 How saveToStorage works internally: It calls
                    JSON.stringify(value) to convert the array to a string, then calls
                    localStorage.setItem(key, stringified). It's wrapped in a try/catch so it gracefully
                    handles cases like private browsing where storage might be disabled.


                
Key Concepts

                

                    localStorage persistence
                    JSON.stringify
                    Modular functions (called after every mutation)
                


                
Likely Instructor Questions

                

                    

                        
Why is this called after every
                                add/edit/delete?▾

                        
products is just a variable living in memory (RAM). The
                            moment the page is closed or refreshed, memory is cleared. localStorage is the only way to
                            keep data across sessions, so we sync it after every change.

                    

                    

                        
Why does saveToStorage need
                                JSON.stringify?▾

                        
localStorage only accepts strings. If you try to store an array directly,
                            JavaScript converts it to something like "object Object" which is useless. JSON.stringify
                            properly converts it to '[{"id":"abc","title":"Laptop",...}]'.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 03
                            Build the filtered + sorted list
                        

                        
filter() · spread · sort() · normalizeText · sortBy

                    

                

                
▾

            

            

                
What it does

                
Returns a brand-new array (never touching the original products) that
                    applies: search filter, category filter, and sort order — based on what the user has selected in the
                    UI controls.


                
The Code

                
function getVisibleProducts() {
  const search   = Utils.normalizeText(els.searchInput.value);
  const category = els.categoryFilter.value;
  const sortBy   = els.sortSelect.value;

  // filter() — only keep products that match both conditions
  let result = Utils.customFilter(products, (p) => {
    const matchesSearch   = Utils.normalizeText(p.title).includes(search);
    const matchesCategory = category === 'All' || p.category === category;
    return matchesSearch && matchesCategory;
  });

  // sort the filtered copy (sortBy clones with spread internally)
  switch (sortBy) {
    case 'price-asc':  result = Utils.sortBy(result, 'price', 'asc');  break;
    case 'price-desc': result = Utils.sortBy(result, 'price', 'desc'); break;
    case 'newest':     result = Utils.sortBy(result, 'createdAt', 'desc'); break;
    case 'oldest':     result = Utils.sortBy(result, 'createdAt', 'asc'); break;
  }
  return result;
}


                
Utils Functions Used

                

                    

                        Utils.normalizeText(str)
                        Trims and lowercases a string so "MOUSE" matches "mouse".
                    

                    

                        Utils.customFilter(array, callback)
                        Custom implementation of Array.filter(). Returns new array of items
                            where callback returns true.
                    

                    

                        Utils.sortBy(array, key, direction)
                        Returns a NEW sorted array by a given key ('asc' or 'desc'). Uses spread
                            to clone first.
                    

                


                
Key Concepts

                

                    filter()
                    sort()
                    Spread operator (clone before sort)
                    Never mutate original array
                    Case-insensitive search
                


                
Likely Instructor Questions

                

                    

                        
Why not sort the products array
                                directly?▾

                        
Because sort() mutates the array in place. If you sorted
                            products directly, the order would change permanently. Next time the user
                            clears the filter, the original insertion order would be gone. By spreading into a new array
                            first ([...array] inside sortBy), the original order is always preserved.

                    

                    

                        
Why use normalizeText before
                                comparing?▾

                        
Without it, searching "laptop" would NOT match a product titled "Laptop"
                            because JavaScript string comparison is case-sensitive. normalizeText
                            lowercases and trims both the search input and the product title before comparing, so the
                            match works regardless of case.

                    

                    

                        
How does customFilter know which
                                products to keep?▾

                        
It runs the callback function on each product. If the callback returns
                            true, the product is kept. Both conditions (matchesSearch AND
                            matchesCategory) must be true. If the search box is empty, normalizeText
                            returns "" and every title includes "", so all products pass that check.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 04
                            Render the products table
                        

                        
forEach · destructuring · template literals · formatCurrency ·
                            formatDate

                    

                

                
▾

            

            

                
What it does

                
Clears the table, then rebuilds it row by row from the visible products. If the
                    list is empty, it shows an "No Products Found" message instead.


                
The Code

                
function renderProducts() {
  const visible = getVisibleProducts();
  els.tableBody.innerHTML = '';
  els.emptyState.classList.toggle('d-none', visible.length > 0);

  Utils.customEach(visible, (product) => {
    // Destructuring — extract fields cleanly instead of product.title etc.
    const { id, title, price, category, stock, createdAt } = product;

    // Template literal — build the HTML row as a string
    const row = `
      <tr>
        <td>${title}</td>
        <td>${Utils.formatCurrency(price)}</td>
        <td>${category}</td>
        <td>${stock}</td>
        <td>${Utils.formatDate(createdAt)}</td>
        <td>
          <button data-action="edit"   data-id="${id}">Edit</button>
          <button data-action="delete" data-id="${id}">Delete</button>
        </td>
      </tr>
    `;
    els.tableBody.insertAdjacentHTML('beforeend', row);
  });
}


                
Utils Functions Used

                

                    

                        Utils.customEach(array, callback)
                        Custom forEach. Runs the callback for every item. Does not return
                            anything.
                    

                    

                        Utils.formatCurrency(value)
                        Formats a number as "$12.99" using Intl.NumberFormat. Returns "—" for
                            invalid values.
                    

                    

                        Utils.formatDate(input)
                        Converts an ISO date string to "DD/MM/YYYY" format for display.
                    

                


                
Key Concepts

                

                    Object destructuring
                    Template literals (backticks + ${})
                    forEach / customEach
                    insertAdjacentHTML
                    Event delegation (data-action, data-id)
                


                
Likely Instructor Questions

                

                    

                        
What does destructuring save you from
                                writing?▾

                        
Without destructuring you'd write product.title,
                            product.price, product.category every time. With
                            const { id, title, price, ... } = product, you unpack those fields once into
                            local variables and use them directly. Cleaner and less repetitive.

                    

                    

                        
Why use data-action and data-id on the
                                buttons?▾

                        
This is called event delegation. Instead of attaching a separate click
                            listener to every single button (which would need to be re-attached every time the table
                            re-renders), one listener on the table body reads btn.dataset.action and
                            btn.dataset.id to know which product to edit or delete. It's more efficient and
                            works for dynamically added rows.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 05
                            Update the statistics section
                        

                        
reduce() · find() · formatCurrency

                    

                

                
▾

            

            

                
What it does

                
Computes three dashboard stats: total count of products, total inventory value
                    (every product's price × stock, all summed), and the title of the most expensive product.


                
The Code

                
function updateStats() {
  els.totalProducts.textContent = products.length;

  // reduce() — accumulate a single number from the whole array
  const inventoryValue = Utils.customReduce(
    products,
    (sum, p) => sum + p.price * p.stock,
    0   // initial value
  );
  els.totalInventoryValue.textContent = Utils.formatCurrency(inventoryValue);

  if (products.length === 0) {
    els.mostExpensiveProduct.textContent = '—';
    return;
  }

  // reduce() again — find the highest price value
  const maxPrice = Utils.customReduce(
    products,
    (max, p) => (p.price > max ? p.price : max),
    -Infinity
  );

  // find() — get the product object whose price equals maxPrice
  const expensive = Utils.customFind(products, (p) => p.price === maxPrice);
  els.mostExpensiveProduct.textContent = expensive ? expensive.title : '—';
}


                
Utils Functions Used

                

                    

                        Utils.customReduce(array, callback, initial)
                        Custom reduce(). Accumulates a single value across all array items using
                            the callback.
                    

                    

                        Utils.customFind(array, callback)
                        Custom find(). Returns the first item where callback returns true.
                            Returns undefined if none found.
                    

                    

                        Utils.formatCurrency(value)
                        Formats a number as currency like "$1,234.56".
                    

                


                
Key Concepts

                

                    reduce() — sum & max
                    find() — get matching object
                    -Infinity as initial max value
                    Guard clause (empty array)
                


                
Likely Instructor Questions

                

                    

                        
Explain reduce() in plain
                                terms.▾

                        
reduce() works like a running total. You give it a starting value (here
                            0) and a function that says "given what I have so far, and the current item,
                            what's the new total?". For the inventory: start at 0, then for each product add
                            price × stock. After the last product, you have the total value. It reduces an
                            entire array down to one single value.

                    

                    

                        
Why use -Infinity as the initial value
                                for the max price?▾

                        
If we started at 0, a product with price $0 would never "win" as the
                            maximum. By starting at -Infinity, even a $0 product is guaranteed to be larger, so it
                            correctly becomes the current max in the first iteration.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 06
                            Validate the form
                        

                        
some() · every() · isRequired · isPositiveNumber ·
                            isNonNegativeInteger

                    

                

                
▾

            

            

                
What it does

                
Checks all four form fields and returns an errors object. Each key in the object
                    is a field name and its value is an error message string. An empty object means everything is valid.
                    The form submit handler reads this to decide whether to proceed.


                
The Code

                
function validateForm({ title, price, category, stock }) {
  const errors = {};

  if (!Utils.isRequired(title))
    errors.title = 'Title is required.';
  if (!Utils.isPositiveNumber(price))
    errors.price = 'Price must be a positive number.';
  if (!Utils.customSome(CATEGORIES, (c) => c === category))
    errors.category = 'Please choose a category.';
  if (!Utils.isNonNegativeInteger(stock))
    errors.stock = 'Stock must be 0 or a positive whole number.';

  return errors;
}

// In the submit handler — uses customEvery to check all errors are absent:
const isValid = Utils.customEvery(Object.keys(errors), (k) => !errors[k]);


                
Utils Functions Used

                

                    

                        Utils.isRequired(value)
                        Returns true if value is not null/undefined and not an empty/whitespace
                            string.
                    

                    

                        Utils.isPositiveNumber(value)
                        Returns true if the value converts to a number that is greater than 0
                            (not NaN).
                    

                    

                        Utils.isNonNegativeInteger(value)
                        Returns true if the value is an integer ≥ 0. Rejects decimals and
                            negatives.
                    

                    

                        Utils.customSome(array, callback)
                        Custom some(). Returns true if at least one item passes the callback
                            test.
                    

                    

                        Utils.customEvery(array, callback)
                        Custom every(). Returns true only if ALL items pass the callback test.
                            Used to confirm no errors exist.
                    

                


                
Key Concepts

                

                    some() — at least one passes
                    every() — all pass
                    Errors object pattern
                    Parameter destructuring
                    Validation utilities
                


                
Likely Instructor Questions

                

                    

                        
What's the difference between some() and
                                every()?▾

                        
some() returns true if AT LEAST ONE item passes
                            the test — like an OR gate. every() returns true only if ALL items
                            pass — like an AND gate. Here, customSome checks if the chosen category is one
                            of the valid options. customEvery checks that no error keys have values.

                    

                    

                        
Why does validateForm take a
                                destructured parameter?▾

                        
The function signature ({ title, price, category, stock })
                            is parameter destructuring. The caller passes a single object (formData), and the function
                            unpacks the four fields it needs directly. It's cleaner than passing four separate arguments
                            or writing formData.title inside the function.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 07
                            Add a new product
                        

                        
Spread operator · generateId · modular functions

                    

                

                
▾

            

            

                
What it does

                
Builds a complete product object by combining an auto-generated ID and timestamp
                    with the form data, then adds it to the products array (without mutating it), and updates
                    everything: storage, table, and stats.


                
The Code

                
function addProduct(formData) {
  const newProduct = {
    id:        Utils.generateId(),        // auto unique ID
    createdAt: new Date().toISOString(), // timestamp
    ...formData,  // spread: copies title, price, category, stock
  };

  products = [...products, newProduct]; // spread: new array with item appended

  persistProducts();  // save to localStorage
  renderProducts();   // update the table
  updateStats();      // update the stats cards
}


                
Utils Functions Used

                

                    

                        Utils.generateId()
                        Creates a unique string ID using Date.now() in base-36 + random
                            characters. Guarantees uniqueness.
                    

                


                
Key Concepts

                

                    Spread in object literal (...formData)
                    Spread to append to array ([...products, newProduct])
                    Auto-generated ID
                    ISO timestamp
                    Modular function chaining
                


                
Likely Instructor Questions

                

                    

                        
What does ...formData do inside the
                                object literal?▾

                        
The spread operator copies all key-value pairs from formData
                            into the new object. So if formData is
                            { title: "Laptop", price: 999, category: "Electronics", stock: 5 }, the new
                            product will contain all those fields plus id and createdAt.

                    

                    

                        
Why use [...products, newProduct]
                                instead of products.push()?▾

                        
push() mutates the original array in-place. The spread
                            creates a brand new array with the new item added. This immutable approach is a safer
                            pattern — the original array reference isn't changed unexpectedly, which avoids subtle bugs.
                            It's also the approach used in modern frameworks like React.

                    

                    

                        
How does generateId() ensure
                                uniqueness?▾

                        
Date.now().toString(36) gives a timestamp in base-36
                            (milliseconds since 1970, very unlikely to repeat). It then appends
                            Math.random().toString(36).slice(2, 8) — 6 random characters. The combination
                            of time + randomness makes a collision virtually impossible.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 08
                            Edit an existing product
                        

                        
findIndex() · spread to merge objects

                    

                

                
▾

            

            

                
What it does

                
Finds the product being edited (by its ID stored in editingId),
                    creates a merged copy that preserves the original id and createdAt while
                    updating the form fields, then replaces it in the array.


                
The Code

                
function saveEdit(formData) {
  // findIndex() — returns the position of the product in the array
  const idx = Utils.customFindIndex(products, (p) => p.id === editingId);
  if (idx === -1) return; // safety guard

  // Spread to MERGE: old values first, new form values override them
  products[idx] = { ...products[idx], ...formData };

  persistProducts();
  renderProducts();
  updateStats();
}


                
Utils Functions Used

                

                    

                        Utils.customFindIndex(array, callback)
                        Custom findIndex(). Returns the index of the first item where callback
                            returns true. Returns -1 if not found.
                    

                


                
Key Concepts

                

                    findIndex() — returns position
                    Spread to merge two objects
                    Preserving id and createdAt
                    Guard clause (idx === -1)
                


                
Likely Instructor Questions

                

                    

                        
Why spread the old product first in the
                                merge?▾

                        
The spread operator is applied left to right, and later keys override
                            earlier ones. By writing { ...products[idx], ...formData }, the old product's
                            id and createdAt are copied first, then formData's
                            title, price, category, and stock
                            overwrite the old versions. If we only spread formData, we'd lose the
                            id and createdAt fields entirely.

                    

                    

                        
What's the difference between find() and
                                findIndex()?▾

                        
find() returns the actual item (the product object).
                            findIndex() returns the position number (e.g., 2 meaning the third slot). Here
                            we need the index because we want to replace that specific slot:
                            products[idx] = .... If we used find(), we'd get the object but wouldn't know
                            where in the array to put the updated version.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 09
                            Delete a product (with Undo)
                        

                        
findIndex() · splice() · array destructuring · undo toast

                    

                

                
▾

            

            

                
What it does

                
Removes a product from the array, but saves a copy of the removed item and its
                    original index in lastDeleted. This enables the Undo feature. Then shows a toast
                    notification at the bottom of the screen.


                
The Code

                
function deleteProduct(id) {
  const idx = Utils.customFindIndex(products, (p) => p.id === id);
  if (idx === -1) return;

  // splice(idx, 1) removes 1 item at idx, returns it in an array
  // Array destructuring: [removed] pulls the first element out
  const [removed] = products.splice(idx, 1);

  // Save removed product + its index for Undo
  lastDeleted = { product: removed, index: idx };

  persistProducts();
  renderProducts();
  updateStats();

  const toast = bootstrap.Toast.getOrCreateInstance(els.undoToast);
  toast.show();
}


                
Utils Functions Used

                

                    

                        Utils.customFindIndex(array, callback)
                        Returns the index (position number) of the first matching item. Returns
                            -1 if not found.
                    

                


                
Key Concepts

                

                    splice(idx, 1) — removes and returns
                    Array destructuring [removed]
                    Saving index for undo
                    Bootstrap Toast API
                


                
Likely Instructor Questions

                

                    

                        
What does splice(idx, 1) do
                                exactly?▾

                        
splice(startIndex, deleteCount) modifies the array in place.
                            splice(idx, 1) says: "starting at position idx, remove 1 item." It returns an
                            array of the removed items. So if idx is 2, it removes the item at index 2 and
                            returns [thatItem].

                    

                    

                        
Why does the destructuring say [removed]
                                instead of just removed?▾

                        
splice() always returns an ARRAY of removed items, even when
                            only one is removed. const [removed] = ... is array destructuring — it unpacks
                            the first element from that returned array into the variable removed. Without
                            the brackets, removed would be the whole array [product], not the
                            product itself.

                    

                    

                        
Why save the index and not just the
                                product?▾

                        
When the user clicks Undo, we want to restore the product to its exact
                            original position in the list. If we only saved the product, we could only push it to the
                            end. Saving the index lets us use splice(index, 0, product) to insert it back
                            precisely where it was.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 10
                            Undo the last delete
                        

                        
Object destructuring · splice() to insert

                    

                

                
▾

            

            

                
What it does

                
If a product was recently deleted, puts it back at its original position using
                    splice with 0 as the delete count (meaning "just insert, don't remove"). Then clears lastDeleted so
                    it can't be undone twice.


                
The Code

                
function undoDelete() {
  if (!lastDeleted) return; // nothing to undo

  // Object destructuring — extract both fields from lastDeleted at once
  const { product, index } = lastDeleted;

  // splice(index, 0, product) = "at position index, delete 0, insert product"
  products.splice(index, 0, product);

  lastDeleted = null; // clear so it can't be undone again

  persistProducts();
  renderProducts();
  updateStats();
}


                
Utils Functions Used

                

                    

                        Utils.saveToStorage (via persistProducts)
                        Called indirectly through persistProducts() to sync the restored array
                            to localStorage.
                    

                


                
Key Concepts

                

                    Object destructuring
                    splice(index, 0, item) — insert without removing
                    Guard clause (!lastDeleted)
                    State reset (lastDeleted = null)
                


                
Likely Instructor Questions

                

                    

                        
Explain splice(index, 0, product) step
                                by step.▾

                        
The three arguments are: (1) where to start — the original index, (2) how
                            many items to delete — 0, meaning don't remove anything, just insert, (3) what to insert —
                            the product object. So it slots the product back in exactly where it was without disturbing
                            anything else.

                    

                    

                        
Why set lastDeleted = null after
                                undoing?▾

                        
If we didn't clear it, the user could click Undo again and add duplicate
                            copies of the product. Setting it to null means the guard at the top
                            (if (!lastDeleted) return) will stop any further undo attempts until the next
                            delete.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 11
                            Export products.json
                            BONUS
                        

                        
JSON.stringify · Blob · Object URL · programmatic download

                    

                

                
▾

            

            

                
What it does

                
Converts the products array to a formatted JSON string, wraps it in a Blob
                    (in-memory file), creates a temporary browser URL for it, then triggers a download by
                    programmatically clicking a hidden anchor tag.


                
The Code

                
function exportProducts() {
  // JSON.stringify(value, replacer, indent) — pretty-printed with 2 spaces
  const json = JSON.stringify(products, null, 2);

  // Blob — creates an in-memory file-like object
  const blob = new Blob([json], { type: 'application/json' });

  // Object URL — temporary URL pointing to the blob
  const url = URL.createObjectURL(blob);

  // Programmatic download via hidden anchor tag
  const a = document.createElement('a');
  a.href = url;
  a.download = 'products.json';
  a.click();

  URL.revokeObjectURL(url); // clean up memory
}


                
Key Concepts

                

                    JSON.stringify with indentation
                    Blob API
                    URL.createObjectURL
                    Programmatic anchor click
                    Memory cleanup (revokeObjectURL)
                


                
Likely Instructor Questions

                

                    

                        
What is a Blob?▾

                        
A Blob (Binary Large Object) is a file-like object that lives in memory
                            in the browser. It's not a real file on the disk yet, but it can hold data (text, binary)
                            and be given a MIME type. Here we create one holding our JSON string with the type
                            "application/json".

                    

                    

                        
Why call revokeObjectURL after
                                clicking?▾

                        
createObjectURL allocates memory for the URL. If we never
                            revoke it, that memory leaks — it stays allocated for the whole session. Calling
                            revokeObjectURL immediately after the click is fine because the download is
                            already queued in the browser's download manager.

                    

                

            

        


        
        

            

                

                    

                    

                        

                            EX 12
                            Dark mode toggle
                            BONUS
                        

                        
Ternary operator · data-bs-theme · saveToStorage

                    

                

                
▾

            

            

                
What it does

                
Flips Bootstrap's built-in dark mode by toggling the data-bs-theme
                    attribute on the HTML element between "light" and "dark". Saves the choice to localStorage so it
                    persists on next visit. Also swaps the moon/sun icons on the button.


                
The Code

                
function toggleTheme() {
  const current = document.documentElement.getAttribute('data-bs-theme');

  // Ternary operator — one-line if/else to flip the value
  const next = current === 'dark' ? 'light' : 'dark';

  document.documentElement.setAttribute('data-bs-theme', next);
  Utils.saveToStorage(THEME_KEY, next); // persist the choice

  // Toggle icon visibility based on current theme
  els.themeToggle
    .querySelector('.theme-icon-light')
    .classList.toggle('d-none', next === 'dark');
  els.themeToggle
    .querySelector('.theme-icon-dark')
    .classList.toggle('d-none', next !== 'dark');
}


                
Utils Functions Used

                

                    

                        Utils.saveToStorage(key, value)
                        Saves the theme string ("light" or "dark") to localStorage so it's
                            remembered on next visit.
                    

                


                
💡 The flash-prevention script in <head>: Before the page
                    fully renders, a small inline script runs and reads the saved theme from localStorage. It applies
                    data-bs-theme before the CSS paints, preventing a visible flash of the wrong color mode
                    on load.


                
Key Concepts

Likely Instructor Questions

                

                    

                        
How does Bootstrap know to switch colors
                                without writing CSS?▾

                        
Bootstrap 5 has CSS custom properties (variables) defined for both light
                            and dark themes. When data-bs-theme="dark" is set on the root
                            <html> element, Bootstrap's stylesheet activates its dark-mode variable
                            values for all components automatically.

                    

                    

                        
Why is the theme script in <head>
                                and not in script.js?▾

                        
Scripts at the bottom of <body> run after the page has already been
                            painted. If theme-switching happened there, the user would see a flash of the default light
                            theme first. The <head> script runs before any rendering, so the correct theme is
                            applied immediately on page load.
